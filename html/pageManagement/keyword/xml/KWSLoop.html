LOOP
com.perfectoMobile.page.keyWord.step.spi.KWSAttribute
The Loop keyword allows you to loop over a number of iterations, a set of data from a data table or a set of elements found on the page. Each iteration of a loop will call the named function. During a loop, the BREAK keyword step can be used in the function to break out of the loop
Attribute Overrides
Name:In this case, name is only used for reporting purposes and serves no purpose as far as execution is concerned
Parameters
The loop takes 2 parameters
1:The loop iterator. The iterator can be in 1 of 3 forms
  • If specified as an integer, the loop will iterator the specified amount of times before exiting
  • If specified using the data tag then the value in the brackets is the table name to loop over. data{authData} will loop over all records in the authData table. If that value inside of the brackets begins with a period then that indicates that we are looping over data tables inside of the page data supplied to the test. data{.authData.users} will cycle over the records in the users table as part of the authData record passed to the test
  • If specified as _CONTEXT.ELEMENT then the current context element will be used. This value can be used for nested loops over elements
  • If none of the other conditions are met then this value used as the page element name. The descriptor will be extracted and the values looked up and iterated over. During the loop, the current selected element will be stored as _CONTEXT.ELEMENT for use in your scripts. Along with the value, the current index will be stored as _CONTEXT.ELEMENT.INDEX. Both of these names can be referenced directly from your scripts to represent the element and index
2:The name of the function to call for each iteration of a loop.
Examples
Loop 10 times calling the changeUser function
Iterate over all records in a data table name authData
Iterate over all records in a data table name authData passed in the userType record to this test case
This will loop over all elements defined by the page element name RECORDS on the page USERS
