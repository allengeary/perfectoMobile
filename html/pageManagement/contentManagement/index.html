Content Management
com.perfectoMobile.content.ContentManager
Localization for a multi-lingual application is every automation engineer’s nightmare. Internationalization and localization are means of adapting computer software to different languages however automated tests need to be created to handle the different languages. Content Management is the process of adapting internationalized software for a specific region or language by adding locale-specific components and translating text for use by the automated test scripts. The engine allows for on the fly content replacement for lookups and verification make multi-lingual support easy to implement.
Usage
When defining a lookup there are a few things to consider.
key only:If you key name provided for lookup contains the name of the key with nothing else, then the value will be located using the first lookup columns value
key{columnName}Using the key followed by the lookup name in curly brackets then that fields value will be used
key{#num}Using the key followed by the pound sign (#) and the index in curly brackets then the fields value indicated by the index will be used
XML Configuration
When defining content using Excel you must provide a key column and lookup columns. They can exist in any position and will be defined.
key:The key column is used to locate the specific row for the lookup value. The key column is case sensitive and must be unique throughout the column.
lookup:The lookup columns(s) are the columns that contain the value for replacement. You must provide a minimum of 1 lookup column.
